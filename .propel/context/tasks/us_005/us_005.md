## Story ID
   * ID: US_005

## Story Title
   * Metric tiles, progress bar & sparklines

## Description
  * As an observer, I want metric tiles (epoch, loss, accuracy), a progress bar and sparklines so that I can quickly understand training behavior.

## Acceptance Criteria
  1. Given simulation epoch updates, When epoch_complete event is received, Then metric tiles display new values and the progress bar adjusts to reflect epoch/totalEpochs percentage.
  2. Given metric streaming, When updates are frequent, Then UI throttles rendering to maintain 60fps while ensuring no more than 5 updates per second on visual components.
  3. Given visual updates, When a critical metric threshold is reached (configurable), Then an aria-live polite announcement is made and visual highlight is applied.
  4. Given sparkline rendering, When points exceed maxPoints, Then the oldest points are removed and performance remains within budget.

## Edge Cases
   * How to handle NaN or undefined metric values? (Display "—" and log the occurrence; do not crash.)
   * How to render on very small viewports? (Sparklines collapse to summarized numeric with mini-trend indicator.)
