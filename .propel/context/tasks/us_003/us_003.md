## Story ID
   * ID: US_003

## Story Title
   * Landing shell and control surface

## Description
  * As an end-user, I want a clear landing shell and control surface (Start, Pause/Resume, Reset, Load Demo Data) so that I can operate the simulation demo easily.

## Acceptance Criteria
  1. Given the application is loaded, When viewing on any supported viewport, Then header, description, hero region, and controls are visible and follow semantic HTML (header, nav, main, footer) and are keyboard accessible.
  2. Given the simulation is idle, When the page loads, Then Start button is enabled and Pause/Resume and Reset are disabled as appropriate.
  3. Given a running simulation, When the user clicks Pause, Then the engine receives a pause command and control state updates to show Resume.
  4. Given a user selects "Load Demo Data", When selection is confirmed, Then the control surface emits the event dataset_selected with the chosen preset.

## Edge Cases
   * What happens when controls are activated while the engine is initializing? (Queue command and show "Initializing" status; disable conflicting controls.)
   * How to handle slow rendering on low-powered devices? (Controls remain responsive; visual updates may be throttled — document behavior.)
