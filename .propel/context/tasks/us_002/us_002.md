## Story ID
   * ID: US_002

## Story Title
   * CI/CD pipeline & quality checks

## Description
  * As a DevOps/Engineer, I want CI jobs for build, test, lint, accessibility, and a Lighthouse performance job so that each change is validated before merge.

## Acceptance Criteria
  1. Given a new PR, When CI is triggered, Then the pipeline runs steps: install, lint, unit tests, accessibility audit, and Lighthouse performance job and reports statuses to the PR within the CI run.
  2. Given the Lighthouse job result, When a metric regresses beyond configured budget thresholds, Then the pipeline marks the run as failed and posts the report link.
  3. Given transient network errors during install, When retry policy is enabled, Then the install step retries up to 3 times with backoff before failing.

## Edge Cases
   * How to handle flaky tests? (Flag flaky tests, mark job as unstable, create ticket; do not block merge until triaged.)
   * What if CI quota is exhausted? (Fail fast with clear error and fallback instructions.)
