## Story ID
   * ID: US_001

## Story Title
   * Project scaffold & developer environment

## Description
  * As a developer, I want a standardized project scaffold with developer scripts and linters so that the team can begin feature work quickly and consistently.

## Acceptance Criteria
  1. Given a repository clone and Node.js LTS installed, When the developer runs `npm run setup`, Then dependencies install and environment variables sample file (.env.example) is created and instructions printed within 10 minutes.
  2. Given the environment is set up, When `npm run dev` is executed, Then the dev server starts and serves the app at localhost:3000 with HMR enabled.
  3. Given the codebase, When `npm run lint` is executed, Then ESLint and Prettier checks run and return exit code 0 or a reproducible list of fixable issues.
  4. Given accessibility linting enabled, When `npm run lint:accessibility` runs, Then accessibility warnings are reported per eslint-plugin-jsx-a11y rules.

## Edge Cases
   * What happens when the developer has an incompatible Node version? (Fail with message recommending exact supported version and link to nvm instructions.)
   * What if the package registry is blocked? (Support alternate registry env var and documented proxy instructions.)
