## Story Title
   * Return Lockout Response with Non-enumeration

## Description
  * As a security engineer, I want the authentication API to return a configured lockout status (default 423 Locked) with a non-enumerating generic message and consistent response timing for locked accounts, so that attackers cannot determine account existence or lockout status while legitimate users receive a predictable, auditable response.

## Acceptance Criteria
  * Given an account is marked locked (locked_at / locked_until set), When any client submits a POST /auth/login attempt for that account (with any credentials), Then the API returns the configured lockout HTTP status code (default 423), a generic non-enumerating JSON body (e.g., { "error": "authentication_failed" } or { "message": "If the credentials are correct, access will be restored. Contact support if needed." }), excludes any account-specific details (failed attempts, locked_until), and an audit entry is created server-side (not returned to client).
