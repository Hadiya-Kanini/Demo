## Story ID
   * ID: US_004

## Story Title
   * Deterministic simulation engine core & API

## Description
  * As a product, I want a deterministic, seedable simulation engine with a small public API so that demo behavior is reproducible and controllable.

## Acceptance Criteria
  1. Given a seed and configuration (epochs, speed, noise), When the engine is started twice with the same inputs, Then the sequence of emitted events and metric time series are identical for both runs.
  2. Given engine states Idle, Running, Paused, Completed, When control commands are issued (start, pause, resume, reset), Then the engine transitions state correctly and emits corresponding events (simulation_started, epoch_complete, simulation_paused, simulation_resumed, simulation_reset).
  3. Given a mid-run pause, When resumed, Then epoch counters and RNG state continue such that determinism is preserved for subsequent epochs.
  4. Given invalid configuration (negative epochs), When engine initialized, Then it rejects config with validation error and does not start.

## Edge Cases
   * What if the seed source is the current time? (Allow explicit seed override and document default nondeterministic behavior.)
   * How to handle heavy compute on slower devices? (Provide speed parameter to throttle sim steps; do not block UI thread — offload to worker if required.)
