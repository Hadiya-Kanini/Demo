## Story ID
   * ID: US_009

## Story Title
   * Persistence for settings & optional state

## Description
  * As an end-user, I want optional persistence for my settings and logs in localStorage so that my preferred presets and view options persist between sessions.

## Acceptance Criteria
  1. Given the user enables "Remember settings", When they change presets or UI preferences, Then the changes persist to localStorage under a namespaced key and restore automatically on next load.
  2. Given the user opts out, When toggled off, Then previously persisted values are removed and future sessions do not restore settings.
  3. Given persisted data, When user clicks "Reset settings", Then localStorage entries are cleared and UI resets to defaults.

## Edge Cases
   * What if localStorage quota exceeded? (Gracefully degrade: show warning and fallback to in-memory only.)
   * How to handle corrupted persisted data? (Detect, clear, and log; show user-facing message.)
