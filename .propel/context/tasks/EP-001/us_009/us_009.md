## Story ID: us_009

## Story Title: Integration & E2E Tests for Auth Flows

## Description:
As a QA/Platform Engineer, I want CI-integrated integration and end-to-end test suites that cover login, token usage (including expiry and revocation), RBAC enforcement, logout, and error paths, so that authentication regressions are detected early, security and RBAC guarantees are validated, and PRs cannot degrade auth behavior.

## Acceptance Criteria (Given/When/Then format, 3-5 criteria)
1. Given a deployed test environment with the auth service and a seeded test user, When the integration test for the happy-path login runs, Then the login endpoint responds 200, returns a correctly formatted token (JWT or agreed token format) with TTL claims, and a protected API endpoint accepts the token and returns 200 with expected payload.

2. Given a valid token, When the token is presented to a protected endpoint before expiry, Then the endpoint allows access (200) and logs the token usage; and when the token is presented after simulated expiry, Then the endpoint returns 401 with a standardized error code/message without exposing sensitive internals.
