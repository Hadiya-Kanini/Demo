## Story ID: us_001

## Story Title: Auth Spike: Schema & Token Contract

## Description: As a backend engineer, I want a time-boxed spike to discover the EP-DATA schema, define the token format and TTL, and produce a precise API contract for /auth/login and /auth/logout, so that engineering can implement secure, interoperable authentication endpoints and agree on token lifecycle and revocation before development begins.

## Acceptance Criteria (Given/When/Then format)
1. Given the spike is started with access to EP-DATA and key stakeholders, when the spike completes (time-box: 3 full working days), then deliver a single artifact containing:
   - EP-DATA schema mapping for authentication (list of tables/fields used by auth),
   - a recommended token model (access and refresh token format, signing algorithm, key management notes),
   - recommended TTL values with rationale,
   - an OpenAPI v3 contract fragment for POST /auth/login and POST /auth/logout (requests, responses, status codes, error payloads),
   - and 3 example JWTs or token samples (signed or mocked) and expected validation rules.

2. Given access to EP-DATA schema and sample records, when analysing user identity and session storage, then produce a field-level mapping that identifies the canonical authentication fields required (minimum: user_id, canonical_email
