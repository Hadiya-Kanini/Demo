## Story ID: us_005

## Story Title: RBAC Middleware and Role Redirect

## Description: As a backend engineer (implementing for authenticated users), I want role-based authorization middleware for protected endpoints and post-login redirect logic to role-specific dashboards, so that users are only allowed access to resources appropriate for their role and are directed to the correct dashboard immediately after login.

## Acceptance Criteria (Given/When/Then format, 3-5 criteria)
1. Given a protected endpoint configured to allow roles [Admin, Manager], When a request contains a valid access token (JWT or session) with the "roles" claim including "Admin", Then the request is allowed to proceed and the endpoint returns the expected 2xx response.

2. Given a request to a protected endpoint with no token or an expired/invalid token, When the middleware runs, Then the system responds with HTTP 401 Unauthorized and a JSON body with a non-revealing error message (e.g., "Authentication required") and logs an authentication failure event.
