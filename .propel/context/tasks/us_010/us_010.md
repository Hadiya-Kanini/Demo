## Story ID
   * ID: US_010

## Story Title
   * Telemetry adapter & opt-in hooks

## Description
  * As a product owner, I want no-op telemetry hooks and an opt-in toggle so that analytics can be enabled without shipping telemetry by default.

## Acceptance Criteria
  1. Given default configuration, When app runs, Then telemetry adapter is a no-op and no network telemetry calls occur.
  2. Given user enables telemetry opt-in, When events are emitted, Then adapter calls the configured telemetry client with minimal payload and respecting privacy (no PII).
  3. Given opt-in state, When toggled off, Then pending telemetry is not transmitted and future events use no-op adapter.

## Edge Cases
   * How to handle intermittent telemetry backends? (Adapter retries with backoff and fails silently; ensure no blocking of UI.)
   * What telemetry shape is required? (Provide example payload and document minimal set.)
